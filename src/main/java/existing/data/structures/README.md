# Use Existing Data Structures

## What this track covers

This track is for **using Java's built-in and standard-library data structures well**.

It focuses on:
- APIs and common operations
- access patterns and performance trade-offs
- choosing the right structure for a problem
- writing practice code that uses standard Java data structures intentionally

## What belongs here

Topics from the sheet that map here:
- ✅ native Java arrays
- ⬜ Java `List` usage
  - ⬜ `ArrayList`
  - ⬜ `LinkedList`
  - ⬜ `CopyOnWriteArrayList`
- ⬜ Java queue usage
  - ⬜ `Deque`
  - ⬜ `PriorityQueue`
  - ⬜ `BlockingQueue`
- ⬜ Java `Map` usage
  - ⬜ `HashMap`
  - ⬜ `LinkedHashMap`
  - ⬜ `TreeMap`
  - ⬜ `SortedMap` / `NavigableMap`
  - ⬜ `ConcurrentHashMap`
  - ⬜ `WeakHashMap`
  - ⬜ `IdentityHashMap`
  - ⬜ `EnumMap`

## What does not belong here

This track does **not** contain handwritten implementations of arrays, linked lists, stacks, queues, trees, graphs, or hash tables. Those belong under `custom-data-structures/`.

## Folder layout

- ✅ `arrays/` — native Java arrays and array-oriented practice
- ⬜ `lists/` — Java list interfaces and implementations
- ⬜ `queues-and-deques/` — queue-related Java APIs
- ⬜ `maps/` — Java map interfaces and implementations

## Notes on tree and graph work

Java does not provide first-class built-in learning structures for most tree and graph topics from the sheet. When standard collections support that work later, document the usage inside those exercises or README sections rather than creating a separate built-in tree or graph folder here.
