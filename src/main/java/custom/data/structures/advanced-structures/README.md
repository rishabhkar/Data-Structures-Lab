# Advanced Structures

## Track

Custom implementation track: `custom-data-structures`

## What this folder covers

This folder is for **specialized data structures that do not fit more naturally into the main arrays, lists, stacks, queues, trees, graphs, or hashing folders**.

## Normalization note

To avoid duplication:
- tree-family structures that are primarily tree implementations belong under `custom-data-structures/trees/`
- graph-family structures belong under `custom-data-structures/graphs/`
- hash-table collision strategies belong under `custom-data-structures/hashing/`
